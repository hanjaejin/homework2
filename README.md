## 1. 앱 개요

**AI 입고검수 앱**은 납품서(거래명세서) 사진을 Google Gemini AI로 자동 분석하여 품목·수량·단가를 추출하고, 실제 입고 수량과 대조한 뒤 결과를 **구글 시트에 자동 저장**하는 모바일 최적화 웹 앱입니다.
해당앱은 코레일유통 편의점인 Storyway에서 물류업체에서 받은 검수확인증을 자동으로 등록하는 앱으로, 매장을 운영하는 점주님들이 연세가 많고 IT기기를 능숙하게 다루기 어려워 글자는 크게 사용은 최대한 간단하게 작성한 데모 입니다.

### 주요 기능

| 기능 | 설명 |
|------|------|
| AI OCR 분석 | 납품서 사진에서 날짜, 전표번호, 거래처코드, 품목 자동 추출 |
| 수량 검수 | 서류상 수량과 실제 수량을 비교하며 차이 즉시 확인 |
| 불일치 강조 | 수량이 다른 품목은 빨간색으로 자동 표시 |
| 구글 시트 저장 | 검수 결과를 클라우드에 자동 기록 |
| 샘플 다운로드 | 테스트용 납품서 샘플 이미지 제공 |
| 모의 테스트 | 사진 없이 검수 화면 동작 확인 가능 |

## 3. 최초 설정 (API 키 등록)

앱을 처음 사용하기 전에 **Google Gemini API 키**를 등록해야 합니다.  
키는 기기에만 저장되며 외부 서버로 전송되지 않습니다.

### 3-1. API 키 발급

1. [Google AI Studio](https://aistudio.google.com/apikey)에 접속합니다.
2. 구글 계정으로 로그인합니다.
3. **"API 키 만들기"** 버튼을 클릭합니다.(현재는 Google API Key만 사용 가능합니다.)
4. 생성된 키(`AIza...` 형태)를 복사합니다.

### 3-2. 앱에 API 키 등록

1. 앱 상단 왼쪽의 **톱니바퀴 아이콘(⚙️)** 을 탭합니다.
2. **설정 팝업**이 열립니다.
3. `Google Gemini API Key` 입력란에 복사한 키를 붙여넣습니다.
4. **"저장 및 닫기"** 버튼을 탭합니다.

> 등록한 키는 브라우저 로컬 스토리지에 저장되므로, 같은 기기·브라우저를 사용하면 다음번 접속 시 재입력이 필요 없습니다.

## 5. 화면별 상세 설명

### 5-1. 홈 화면 (업로드)

앱 실행 시 가장 처음 보이는 화면입니다.

```
┌─────────────────────────────────┐
│  ⚙️  검수 목록              👤  │  ← 헤더
├─────────────────────────────────┤
│                                 │
│   📷  사진 찍기                 │  ← 카메라로 바로 촬영
│                                 │
├─────────────────────────────────┤
│                                 │
│   🖼️  앨범에서 선택             │  ← 저장된 사진 불러오기
│                                 │
├──────────────┬──────────────────┤
│  ⬇️ 샘플다운 │  🟢 모의 테스트 │  ← 테스트 도구
├─────────────────────────────────┤
│  ✅ 밝은 곳에서 영수증이        │
│     잘 보이게 찍어주세요.       │
└─────────────────────────────────┘
```

#### 버튼 설명

| 버튼 | 동작 |
|------|------|
| **사진 찍기** | 카메라 앱을 열어 납품서를 바로 촬영 |
| **앨범에서 선택** | 갤러리에서 저장된 납품서 사진 선택 |
| **샘플다운** | 테스트용 납품서 샘플 이미지(`sample_receipt.jpg`) 다운로드 |
| **모의 테스트** | 더미 데이터로 검수 화면 즉시 체험 |

---

### 5-2. 분석 중 화면 (로딩)

사진 업로드 후 AI가 분석하는 동안 표시되는 화면입니다.

- 주황색 스피너가 회전하며 진행 상태를 메시지로 안내합니다.
- 분석 시간: 네트워크 환경에 따라 **10~30초** 소요
- 메시지 순서:
  1. `사진을 분석하고 있습니다...`
  2. `거의 다 텍스트를 추출했습니다...`
  3. `데이터를 꼼꼼히 확인 중입니다...`

> 분석 중에는 화면을 닫거나 앱을 전환하지 마세요.

---

### 5-3. 검수 화면 (Inspect)

AI 분석이 완료되면 자동으로 이동하는 핵심 화면입니다.

```
┌─────────────────────────────────┐
│  날짜: 2026-05-18  전표: T-001  │
│  거래처: 코레일유통              │
├─────────────────────────────────┤
│  테스트상품 A        서류: 10개 │
│  코드: 880111                   │
│  단가: 2,000원                  │
│        [-]  [ 10 ]  [+]         │  ← 수량 조절
├─────────────────────────────────┤
│  테스트상품 B  ← 빨간 테두리   │  ← 수량 불일치 강조
│  코드: 880222        서류: 5개  │
│  단가: 1,500원                  │
│        [-]  [  4 ]  [+]         │
├─────────────────────────────────┤
│  ← 다시 찍기    💾 시트에 저장  │
└─────────────────────────────────┘
```

#### 상단 정보 카드

| 항목 | 설명 |
|------|------|
| **날짜** | 납품서에서 추출한 날짜 (YYYY-MM-DD) |
| **전표** | 납품서의 전표번호 |
| **거래처** | 거래처 코드 또는 이름 |

#### 품목 카드

각 품목마다 하나의 카드가 표시됩니다.

| 항목 | 설명 |
|------|------|
| **상품명** | 납품서에서 추출한 품목명 |
| **코드** | 상품 코드 |
| **서류: N개** | 납품서에 기재된 수량 (변경 불가) |
| **단가** | 납품서에서 추출한 단가 |
| **수량 조절 (+/-)** | 실제 입고 수량 수정 |

#### 수량 수정 방법

**방법 1 — 버튼 사용**
- `-` 버튼: 수량 1 감소 (0 이하로 내려가지 않음)
- `+` 버튼: 수량 1 증가

**방법 2 — 직접 입력**
- 가운데 숫자 박스를 탭한 후 키패드로 직접 숫자 입력

#### 불일치 표시

서류 수량과 실제 수량이 다를 경우, 해당 카드의 **테두리가 빨간색**으로 변하고 배경이 연분홍색으로 강조됩니다.

#### 하단 버튼

| 버튼 | 동작 |
|------|------|
| **← 다시 찍기** | 홈 화면으로 돌아가 사진 재촬영 |
| **시트에 저장** | 구글 시트에 검수 결과 저장 후 완료 화면으로 이동 |

## 6. 샘플 및 모의 테스트

### 샘플 이미지로 테스트하기

1. 홈 화면에서 **"샘플다운"** 버튼을 탭합니다.
2. `sample_receipt.jpg` 파일이 기기에 다운로드됩니다.
3. **"앨범에서 선택"** 버튼을 탭하여 다운로드된 샘플 이미지를 선택합니다.
4. AI가 샘플 납품서를 분석하여 검수 화면으로 이동합니다.

### 모의 테스트 (사진 없이 체험)

API 키나 사진 없이 검수 화면을 즉시 체험할 수 있습니다.

1. 홈 화면에서 **"모의 테스트"** 버튼(초록색)을 탭합니다.
2. 아래 더미 데이터로 검수 화면이 바로 열립니다.

| 항목 | 값 |
|------|----|
| 날짜 | 오늘 날짜 |
| 전표번호 | TEST-123456 |
| 거래처 | 코레일유통(테스트) |
| 상품 A | 테스트상품 A, 코드: 880111, 수량: 10개, 단가: 2,000원 |
| 상품 B | 테스트상품 B, 코드: 880222, 수량: 5개 → **4개로 조정됨** |

> 상품 B는 서류 수량(5)과 실제 수량(4)이 다르게 설정되어 **불일치 강조 표시**를 확인할 수 있습니다.

## 7. 결과 확인 (구글 시트)

저장된 검수 데이터는 연동된 구글 스프레드시트에서 확인할 수 있습니다.

### 저장되는 데이터 항목

| 컬럼 | 내용 | 예시 |
|------|------|------|
| timestamp | 저장 시각 | 2026. 5. 18. 오후 3:20:00 |
| date | 납품서 날짜 | 2026-05-18 |
| docNumber | 전표번호 | T-123456 |
| customerCode | 거래처코드 | 코레일유통 |
| itemName | 상품명 | 테스트상품 A |
| itemCode | 상품코드 | 880111 |
| originalQty | 서류 수량 | 10 |
| currentQty | 실제 수량 | 9 |
| unitPrice | 단가 | 2000 |
| isDifferent | 수량 불일치 여부 | O (불일치) / X (일치) |

> `isDifferent` 컬럼이 `O`인 행은 실제 수량과 서류 수량이 다른 품목입니다.

## 8. 주의사항 및 팁

### 사진 촬영 팁

- **밝은 곳**에서 촬영하세요. 그림자가 생기면 인식률이 낮아집니다.
- 납품서 전체가 **화면 안에 들어오도록** 촬영하세요.
- 글씨가 흐릿하지 않도록 **초점이 맞은 상태**에서 찍으세요.
- 구겨진 납품서는 최대한 **펴서** 촬영하세요.

### API 키 관리

- API 키는 **다른 사람과 공유하지 마세요.**
- 공용 기기를 사용할 경우 설정에서 API 키를 삭제한 뒤 로그아웃하세요.
- API 키는 서버에 전송되지 않고 현재 기기(브라우저 로컬 스토리지)에만 저장됩니다.

### 네트워크

- AI 분석과 구글 시트 저장 모두 **인터넷 연결이 필요**합니다.
- Wi-Fi 환경이 권장됩니다. 느린 LTE 환경에서는 분석 시간이 길어질 수 있습니다.
- 60초 이상 응답이 없으면 타임아웃 오류가 발생하며 홈으로 돌아갑니다.

## 9. 문제 해결 (FAQ)

### Q. "설정 메뉴에서 Gemini API Key를 먼저 입력해주세요" 메시지가 뜹니다.
**A.** 헤더의 톱니바퀴(⚙️) 버튼을 눌러 Gemini API 키를 입력하고 저장하세요. API 키 발급 방법은 [3-1. API 키 발급](#3-1-api-키-발급) 항목을 참고하세요.

### Q. AI 분석이 완료됐는데 품목 정보가 "확인불가"로 표시됩니다.
**A.** 납품서 이미지가 흐리거나 너무 어둡게 촬영된 경우입니다. 밝은 곳에서 다시 촬영해주세요.

### Q. 분석 중 "오류" 메시지와 함께 홈으로 돌아왔습니다.
**A.** 아래 사항을 확인하세요.
1. 인터넷이 연결되어 있는지 확인
2. API 키가 올바르게 입력되어 있는지 확인
3. 사진 파일이 손상되지 않았는지 확인
4. 잠시 후 다시 시도

### Q. 샘플다운 버튼을 눌렀는데 "샘플 파일을 찾을 수 없습니다" 경고가 뜹니다.
**A.** 앱이 최신 상태가 아닐 수 있습니다. 페이지를 새로고침(F5 또는 강제 새로고침)한 뒤 다시 시도하세요.

### Q. 구글 시트에 데이터가 저장되지 않습니다.
**A.** 저장 자체는 완료 화면이 뜨면 성공한 것입니다. 구글 시트 반영에는 수 초~수십 초 딜레이가 생길 수 있습니다. 잠시 후 구글 시트를 새로고침하여 확인하세요.

### Q. 수량 입력 후 저장 버튼이 보이지 않습니다.
**A.** 화면을 위로 스크롤하면 하단 버튼 영역이 보입니다. 하단 네비게이션 바 위에 버튼이 위치합니다.
